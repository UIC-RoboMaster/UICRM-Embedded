# 舵轮底盘示例 (45°对角线布局)

## 概述

这个示例演示了如何使用舵轮底盘，采用**45°对角线布局**。舵轮底盘需要8个电机：
- 4个驱动电机 (3508)：控制轮子转速
- 4个转向电机 (6020/4310)：控制轮子转向角度，需要具备绝对编码器

## 运动学算法

本实现参考了 [scutrobotlab/RM2010_AGVinfantry](https://github.com/scutrobotlab/RM2010_AGVinfantry) 的算法。

### 轮子布局 (45°对角线配置)

```
        1(前左)          4(前右)
              \        /
               \  w  /
                \  /
                 中心
                /  \
               /    \
        2(后左)          3(后右)
```

四个轮子相对于底盘中心呈45°对角线分布，每个轮子与底盘中心的距离为 R。

### 运动学公式

假设底盘整体速度为：
- x_speed: 前进速度 (沿x轴)
- y_speed: 平移速度 (沿y轴)
- turn_speed: 旋转速度 (角速度)

对于每个轮子，其速度由两部分组成：
1. **平移速度** (Vx, Vy)：与底盘整体速度相同
2. **旋转速度** (Vw)：Vw = turn_speed × R

#### 轮1 (前左，135°位置)
```
V1 = √[(Vy1 - Vw1·cos45°)² + (Vx1 - Vw1·sin45°)²]
θ1 = atan2((Vy1 - Vw1·cos45°), (Vx1 - Vw1·sin45°))
```

#### 轮2 (后左，225°位置)
```
V2 = √[(Vy2 - Vw2·cos45°)² + (Vx2 + Vw2·sin45°)²]
θ2 = atan2((Vy2 - Vw2·cos45°), (Vx2 + Vw2·sin45°))
```

#### 轮3 (后右，315°位置)
```
V3 = √[(Vy3 + Vw3·cos45°)² + (Vx3 + Vw3·sin45°)²]
θ3 = atan2((Vy3 + Vw3·cos45°), (Vx3 + Vw3·sin45°))
```

#### 轮4 (前右，45°位置)
```
V4 = √[(Vy4 + Vw4·cos45°)² + (Vx4 - Vw4·sin45°)²]
θ4 = atan2((Vy4 + Vw4·cos45°), (Vx4 - Vw4·sin45°))
```

其中：
- V1-V4: 各轮子的驱动速度
- θ1-θ4: 各轮子的转向角度
- cos45° = sin45° = 0.707106781 (√2/2)

## 硬件配置

### 电机连接

**驱动电机 (CAN1):**
- 前左驱动 (轮1): 3508, CAN1 ID 0x201
- 前右驱动 (轮4): 3508, CAN1 ID 0x202
- 后左驱动 (轮2): 3508, CAN1 ID 0x203
- 后右驱动 (轮3): 3508, CAN1 ID 0x204

**转向电机 (CAN2):**
- 前左转向 (轮1): 6020, CAN2 ID 0x205
- 前右转向 (轮4): 6020, CAN2 ID 0x206
- 后左转向 (轮2): 6020, CAN2 ID 0x207
- 后右转向 (轮3): 6020, CAN2 ID 0x208

### 底盘参数

在 `boards/components/src/chassis.cpp` 的 `CHASSIS_STEER_WHEEL` case 中，需要根据实际机器人尺寸调整：

```cpp
const float R = 0.4f;  // 底盘中心到各轮子的距离，单位：米
                       // 建议范围: 0.3-0.5米
```

**如何测量 R：**
1. 测量底盘中心到任一轮子中心的直线距离
2. 由于是45°对角线布局，R = √[(轮距/2)² + (轴距/2)²]
3. 如果轮距和轴距相等（正方形布局），则 R = (轮距/2) × √2

## 编译方法

### 前提条件
1. 确保已安装ARM工具链 (arm-none-eabi-gcc)
2. 确保ARM工具链已添加到PATH环境变量

### 编译步骤

```bash
# 1. 创建build目录
mkdir build && cd build

# 2. 配置CMake
cmake -DCMAKE_BUILD_TYPE=Release ..

# 3. 编译舵轮底盘示例
make example_chassis_DJI_Board_TypeC_general_steer_wheel -j4
```

编译成功后，会生成以下文件：
- `example_chassis_DJI_Board_TypeC_general_steer_wheel.elf`
- `example_chassis_DJI_Board_TypeC_general_steer_wheel.bin`
- `example_chassis_DJI_Board_TypeC_general_steer_wheel.hex`

## 控制说明

使用DBUS遥控器控制底盘：

- **通道0 (ch0)**: 左右平移速度 (x方向)
- **通道1 (ch1)**: 前后平移速度 (y方向)
- **通道2 (ch2)**: 旋转速度 (自转)
- **左拨杆 (swl)**: UP或DOWN位置触发紧急停止

## PID参数

### 驱动电机 (3508)
- **模式**: OMEGA (速度控制)
- **PID参数**: kp: 2500, ki: 3, kd: 0
- **减速比**: 19
- **最大输出**: 30000
- **最大积分输出**: 10000

### 转向电机 (6020)
- **模式**: THETA + ABSOLUTE (绝对角度控制)
- **角度环PID**: kp: 30, ki: 0, kd: 1.5
- **速度环PID**: kp: 1500, ki: 1, kd: 0
- **减速比**: 1 (直接输出)
- **最大输出**: 25000

## 45°布局的优势

1. **更好的力矩分布**：四个轮子对称分布，旋转时受力更均匀
2. **更大的旋转力矩**：相比传统矩形布局，45°布局在相同尺寸下有更大的旋转半径
3. **结构紧凑**：可以在有限空间内实现更好的机动性能
4. **全向移动**：支持任意方向的平移和旋转

## 注意事项

1. **转向电机必须使用具备绝对编码器的电机** (如6020、4310)
2. **ABSOLUTE模式**：转向电机使用ABSOLUTE模式，角度范围限制在 [0, 2π]，会自动选择最短路径旋转
3. **参数 R 非常重要**：直接影响旋转性能，必须准确测量
4. **CAN总线分配**：驱动电机和转向电机使用不同CAN总线，避免通信拥堵
5. **速度归一化**：代码会自动限制最大速度，防止电机过载
6. **功率限制**：可根据比赛规则调整功率限制参数

## 调试步骤

1. **静态测试**：
   - 先单独测试每个电机是否正常
   - 检查转向电机零点位置
   - 确认电机ID配置正确

2. **单轮测试**：
   - 逐个轮子测试转向和驱动
   - 确认角度方向正确（顺时针/逆时针）
   - 确认速度方向正确（前进/后退）

3. **整体测试**：
   - 从小速度开始测试
   - 先测试纯平移（x、y方向）
   - 再测试纯旋转
   - 最后测试组合运动

4. **参数调优**：
   - 如果旋转不稳定，调整参数 R
   - 如果转向响应慢，调整转向电机PID
   - 如果驱动力不足，调整驱动电机PID

## 常见问题

**Q: 为什么旋转时底盘会偏移？**
A: 检查参数 R 是否准确，应该等于底盘中心到轮子的实际距离。

**Q: 转向电机抖动？**
A: 可能是ABSOLUTE模式下角度跳变太大，或者PID参数需要调整。

**Q: 某个轮子不转？**
A: 检查电机ID、CAN总线连接、电机模式配置是否正确。
